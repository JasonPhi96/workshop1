Testing API with CloudShell

In this step, you will test your API using CloudShell. AWS CloudShell is a browser-based, pre-authenticated shell that you can launch directly from the AWS Management Console. You can navigate to CloudShell from the AWS Management Console a few different ways. In this workshop, we will use CloudShell terminal available on AWS console.

get URL to evoke your API

  1. Sign in to the API Gateway console
  2. Choose your API
  3. Note your API’s invoke URL. It appears under Invoke URL on the Details page.

    The full URL looks like this: https://vi6vlcxo32.execute-api.eu-east-1.amazonaws.com

  4. Copy your API’s invoke URL.

Test your API

  1. Open the CloudShell console
  2. In the following command replace “https://vi6vlcxo32.execute-api.eu-east-1.amazonaws.com” with your Invoke URL from the previous step to set a variable with the Invoke URL
#DO NOT COPY THIS!!!
#Replace URL with the Invoke URL above
export INVOKE_URL="https://**vi6vlcxo32**.execute-api.eu-west-1.amazonaws.com"
  1. Create or update an item. The command includes a request body with the item’s ID, price, and name.
curl -X "PUT" -H "Content-Type: application/json" -d "{
    \"id\": \"abcdef234\",
    \"price\": 12345,
    \"name\": \"myitem\"
}" $INVOKE_URL/items

Check the result on DynamoDB

  1. Sign in to the [DynamoDB console] (https://console.aws.amazon.com/DynamoDB/)
  2. Select Tables in the left pane
  3. Select your table
  4. Select Items tab
  5. Confirm data from your PUT from previous step Congratulations! You’ve just update the table in DynamoDB with an item.

Get all items

Use the following command to list all items. curl -s $INVOKE_URL/items

Get an item

Use the following command to get an item by its ID. curl -s $INVOKE_URL/items/abcdef234

Delete an item

Use the following command to delete an item. curl -X "DELETE" $INVOKE_URL/items/abcdef234

Get all items to verify that the item was deleted. curl -s $INVOKE_URL/items
